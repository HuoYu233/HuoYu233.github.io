Java 多线程学习笔记

HawYior
2022-12-18

多线程

01线程简介

  • 多任务

本质上是同一时间依旧只做了一件事

  • 多线程

eg.充分利用道路，添加多个车道

多条执行路径，主线程和子线程并行交替执行

02线程创建

  • 继承Thread类
  • 实现Runnable接口
  • 实现Callable接口

Thread

  • 继承Thread类
  • 重写run()方法
  • 调用start()开启线程

线程开启不一定立即执行，由CPU调度

Runnable

  • 定义类实现Runnable接口
  • 实现run()方法
  • 执行线程需要丢入Runnable接口实现类

避免单继承的局限性

龟兔赛跑

Callable

  • 实现Callable接口，需要返回值类型
  • 重写call方法，需要抛出异常
  • 创建目标对象
  • 创建执行服务
  • 提交执行结果
  • 获取结果
  • 关闭服务

Lambda表达式

  • 避免匿名内部类定义过多

  • 简洁代码

  • 函数式编程

函数式接口

  • 任何接口，如果只包含唯一一个抽象方法，就是一个函数式接口
  • 对于函数式接口，可以通过Lambda表达式创建对象

总结

  • 前提是接口为函数式接口
  • 多个参数也可以去掉参数类型，必须全部去掉，并且加上括号

静态代理

static proxy

  • 真实对象和代理对象实现同一个接口
  • 代理对象要代理真实角色

好处

  • 代理对象可以做真实对象不能做的事情
  • 真实对象专注做自己的事情

03线程状态

创建，就绪，阻塞，运行，死亡

new，start()，sleep/wait/同步锁定，执行代码块，线程结束

  • setPriority(int newPriority)
  • sleep(long mills)
  • join()
  • yield()
  • isAlive()

停止线程

  • 建议线程自动停止，不建议死循环

  • 建议使用标志位

  • 不要使用stop()或destroy()等过时方法

线程休眠

sleep(时间) 指定当前线程阻塞的毫秒数

每一个对象都有一个锁，sleep不会释放锁

模拟网络延时：放大问题发生性

线程礼让

让当前正在执行的线程暂停，但不阻塞

让线程从运行状态转为就绪状态

让CPU重新调度，礼让不一定成功

yield()

Join

合并线程，待此线程执行完成后，再执行其他线程，其他线程阻塞

可以想象成插队

线程状态观测

Thread.State

  • NEW
  • RUNNABLE
  • BLOCKED
  • WAITING
  • TERMINATED

线程优先级

用数字表示，范围1-10

  • getPriority()
  • setPriority(int i)

先设置优先级，再启动

也取决于CPU调度

守护（daemon）线程

线程分为用户线程和守护线程

虚拟机必须确保用户线程执行完毕

虚拟机不能等待守护线程执行完毕

如 后台记录操作日志，监控内存，垃圾回收

setDaemon(true) //默认false表示用户线程

03线程同步

多个线程操作同一个资源

  • 并发

线程同步就是一种等待机制，多个需要同时访问对象的线程进入等待池形成队列

  • 队列和锁

安全性

在访问时加入锁机制synchronized

  • 同步方法

弊端：

同步监视器就是this

若将一个大的方法声明为synchronized 将会影响效率

方法里面需要修改的内容才需要锁

锁的太多，浪费资源

  • 同步块

Obj称为同步监视器

可以是任何对象，但是推荐使用共享资源作为同步监视器

  • 死锁

多个线程互相抱着对方需要的资源，然后形成僵持

某一个同步块同时拥有“两个以上对象的锁”时，就可能发生死锁问题

  • 产生死锁的四个必要条件

  • 锁

Lock是显式锁（手动开启和关闭锁 synchronized是隐式锁

Lock只有代码块锁 synchronized有代码块锁和方法锁

Lock锁，JVM花费更少时间调度线程，性能更好

优先使用顺序：Lock>同步代码块>同步方法

04线程协作

生产者消费者模式

  • 管程法

  • 信号灯法

  • 使用线程池

提前创建好多个线程，放入线程池中，使用时直接获取，使用完放回池中

可以避免频繁创建销毁，实现重复利用
