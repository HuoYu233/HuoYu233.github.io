Git与github的基本使用

2023-04-04 教程 737 字

本文建立在读者已经拥有Github账号，本地安装了Git以及绑定了SSH密钥的情况下

基本使用

  • 首先在Github上新建一个仓库(Create a new repository)

  1. 仓库名
  2. 仓库的描述
  3. 仓库类型，个人向一般都是public
  4. 协议

此外还可以选择是否添加README文件和.gitignore文件

前者可以描述项目的主要内容，后者用于对文件的过滤

至此，一个远程的仓库就创建好了

  • 实现本地代码和仓库代码同步

当你创建好仓库后，你可以选择下面的方式进行代码托管

  1. 在本地初始化git仓库后，绑定远程仓库

首先我在桌面上创建一个demo文件夹，里面有一个hhh.txt文件

在目录下执行git init操作

此时文件夹多出了一个**.git**隐藏文件夹

推荐常使用git status命令查看仓库状态

此时我们执行这个命令

大概的意思是，在master分支下，没有文件可以提交；此外还提醒我们hhh.txt文件没有上传

此时我们执行git add hhh.txt

如果接下来没有结果，那么恭喜你

没有结果就是最好的结果

此时再使用git status查看状态

发现hhh.txt “to be committed” ，也就是这个文件已经放在了缓存区（防止你手滑上传了一些文件），你需要再次确认是否上传这个文件

于是我们输入git commit hhh.txt -m "第一次提交"

你可能会好奇 “-m” 和后面的“第一次提交”是个啥

如果你有观摩过一些开源项目，会注意到这个

也就是“更新说明”，你可以在commit的时候携带信息，说明这次更新的主要内容

于是乎，文件hhh.txt就上传成功了

接下来就是需要绑定远程仓库

输入git remote add origin 仓库地址

没有结果就是最好的结果

然后我们将我们的文件push到远程仓库

git push -u origin 分支名

打开我们的远程仓库，可以看见

至此，这就是最基本的git使用。这是第一种方式

  1. clone到本地进行push

获取到你的远程仓库地址

输入git clone 仓库地址可能会花费一点时间

此时你clone的文件夹里面就会出现这个项目

进入项目文件夹，添加文件xxx.txt

你可以发现这个文件夹自带.git目录，所以你不需要去初始化和绑定远程仓库

git add xxx.txt

git commit xxx.txt -m "嘻嘻嘻"

git push origin [分支]

即可

创建一个IDEA项目（我这里创建Springboot），勾选create git repository

IDEA中可以一键进行commit/push操作

进行push
